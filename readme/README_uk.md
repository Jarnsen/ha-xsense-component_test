## Огляд
Ця інтеграція Home Assistant робить пристрої X-Sense доступними в розумному домі. Вона базується на оригінальній роботі Theo Snel і встановлюється через HACS.

Рекомендуємо створити окремий обліковий запис X-Sense для Home Assistant і з основного облікового запису поділитися лише підтримуваними пристроями.

## Встановлення
У HACS додайте користувацький репозиторій `https://github.com/Jarnsen/ha-xsense-component_test`, завантажте інтеграцію, виконайте інструкції HACS щодо перезапуску й налаштуйте її з обліковим записом X-Sense для Home Assistant.

## Підтримувані пристрої
Підтримуються базові станції, датчики диму, детектори CO, теплові тривоги, датчики протікання води, гігрометри, датчики дверей і руху, світильники, клавіатури, датчики поштової скриньки, пристрої прослуховування та підтримувані камери, якщо їх повідомляє обліковий запис X-Sense.

Підтверджені сімейства моделей: SBS50, XH02-M, XC01-M, XC04-WX, XS01-M, XS01-WX, XS03-WX, XS0B-MR, SC07-WX, XP0A-MR, SWS51, STH51, STH0A, STH0B, STH0C, SDS0A, SMS0A, SSC0A, SSC0B.

## Сутності та дії
Інтеграція створює сутності лише для даних, які пристрій справді повідомляє. Це можуть бути тривоги, вимкнення звуку, батарея, сигнал, температура, вологість, CO, читабельні часові поля, налаштування камер, LED-перемикачі, а також кнопки тесту, вимкнення звуку й пожежного тренування.

Керування пристроями, спільний доступ, видалення, firmware, облікові записи та платежі залишаються в застосунку X-Sense. Для обговорень використовуйте Discord або форум Home Assistant.

## Приклади автоматизацій
```yaml
automation:
  - alias: "Попередження X-Sense про температуру"
    trigger:
      platform: numeric_state
      entity_id: sensor.xsense_temperature
      above: 30
    action:
      service: notify.notify
      data:
        message: "Температура перевищила 30 градусів!"
```

```yaml
automation:
  - alias: "Тривога протікання води"
    trigger:
      platform: state
      entity_id: binary_sensor.xsense_waterleak
      to: "on"
    action:
      service: notify.notify
      data:
        message: "Виявлено протікання води!"
```

## Додаткові подробиці

### Налаштування облікового запису

Рекомендується використовувати окремий обліковий запис X-Sense для Home Assistant і ділитися з ним лише підтримуваними пристроями, які потрібно бачити в Home Assistant. Інтеграція не додає, не видаляє і не переносить пристрої між домівками. Таке керування пристроями залишається в офіційному застосунку X-Sense.

### Оновлення стану

Інтеграція використовує повідомлення MQTT shadow для швидких змін стану та обережне періодичне опитування хмари для оновлення даних. Стан, отриманий від станції, оновлюється на станції, а стан, отриманий від дочірнього пристрою, оновлюється на цьому пристрої, щоб тривоги й сенсори не залишалися на старих значеннях.

### Доступні сутності

Залежно від моделі можуть відображатися тривоги диму, CO, води, температури, руху та дверей, вимкнення звуку тривоги, завершення строку служби, заряджання, стан нагадування, стан світла та інші діагностичні бінарні сенсори. Сенсори можуть містити батарею, RF або Wi-Fi сигнал, прошивку, температуру, вологість, рівень CO, пікове значення CO, гучність, пороги, зручний для читання час, часовий пояс та іншу діагностику. Перемикачі, вибори та числові значення створюються лише тоді, коли пристрій справді їх підтримує.

### Камери

Підтримувані камери можуть надавати сутність камери, мініатюри, прямий потік, стан підключення та налаштування, узгоджені із застосунком X-Sense. Якщо в Home Assistant доступний шлях WebRTC, інтеграція може використовувати його для відповідного перегляду наживо.

### Усунення несправностей

Якщо сутність відсутня, спершу перевірте в застосунку X-Sense, що пристрій справді показує це значення. Якщо стан залишається застарілим, перезавантажте інтеграцію лише як тимчасову перевірку та додайте до звіту діагностику і відповідні рядки журналу Home Assistant.

### Поведінка пристроїв

- Станції та дочірні пристрої можуть повідомляти різні набори значень. Тому інтеграція не припускає, що кожна станція обов'язково має дочірній пристрій.
- Значення часу перетворюються на читабельний формат, якщо пристрій надсилає час у форматі, який використовує застосунок X-Sense.
- Сутність не створюється, якщо пристрій не повідомляє відповідну можливість. Це запобігає появі оманливих елементів керування в Home Assistant.

### Навантаження на хмару

Інтеграція намагається обережно використовувати API X-Sense. Швидкі зміни беруться з повідомлень MQTT, а хмарні виклики використовуються лише тоді, коли вони потрібні для входу, завантаження пристроїв або оновлення стану.

### Повідомлення про проблему

Повідомляючи про помилку, вкажіть модель пристрою, версію інтеграції, чи видно правильне значення в застосунку X-Sense, і додайте діагностику інтеграції з Home Assistant. Також корисно коротко описати, чи стан ніколи не змінюється, чи змінюється лише після перезавантаження інтеграції.

## Повний довідковий розділ

### Налаштування облікового запису докладно
- Для Home Assistant використовуйте окремий обліковий запис X-Sense.
- З основного облікового запису діліться лише підтримуваними пристроями.
- Сполучення, видалення, спільний доступ і перенесення пристроїв залишаються в застосунку X-Sense.
- Якщо застосунок і Home Assistant виходять один з одного, імовірно використовується той самий обліковий запис.

### Оновлення та навантаження API
- Швидкі зміни стану надходять через повідомлення MQTT shadow.
- Хмарні запити використовуються для входу, завантаження пристроїв і оновлення стану.
- Періодичне опитування є резервом, якщо повідомлення MQTT не надійшло.

### Сутності та дії
- Сутності створюються лише для полів, які X-Sense справді повідомляє.
- Діагностичні значення позначаються як діагностика.
- Тест, вимкнення звуку, пожежне тренування та пробудження камери доступні лише для підтримуваних моделей.

### Довідник щодо камер
- Підтримувані камери можуть надавати сутність камери, мініатюру, живий потік і діагностику.
- Шлях WebRTC використовується лише тоді, коли він доступний у Home Assistant.
- SD-карта, платежі, прошивка та керування обліковим записом залишаються в застосунку X-Sense.

### Контрольний список усунення несправностей
- Звіт про проблему має містити модель, версію інтеграції, діагностику та відповідні журнали.

### Межі
- Інтеграція не додає, не видаляє і не переносить пристрої між домівками.

## Контрольний список пристроїв і сутностей

### Основні сімейства пристроїв
- SBS50: базова станція та стан на рівні станції.
- XS01-WX: Wi-Fi димова сигналізація, включно з обліковими записами без окремого дочірнього пристрою.
- XS01-M, XS03-WX, XS0B-MR: сімейства димових сигналізацій.
- XC01-M, XC04-WX: сімейства сигналізацій CO.
- SC07-WX, XP0A-MR: комбіновані сімейства диму та CO.
- XH02-M: сімейство теплових сигналізацій.
- SWS51: сімейство детекторів витоку води.
- STH51, STH0A, STH0B, STH0C: температура та вологість.
- SDS0A: датчик дверей.
- SMS0A: датчик руху.
- SSC0A, SSC0B: підтримувані камери.

### Поля стану
- Стан тривоги показується, коли X-Sense повідомляє поле тривоги.
- Стан вимкнення звуку показується, коли X-Sense повідомляє поле mute.
- Стан батареї показується, коли пристрій повідомляє дані батареї.
- Значення RF і Wi-Fi показуються, коли пристрій їх повідомляє.
- Компактні часові значення X-Sense перетворюються на читабельні сенсори Home Assistant.

### Керування та звіти
- Перемикачі створюються лише для записуваних налаштувань, які повідомляє X-Sense.
- Кнопки створюються лише для дій, підтримуваних застосунком.
- Керування камерою створюється лише коли API позначає його доступним.
- Звіт про проблему має містити точну модель, версію інтеграції, діагностику, журнали та чи змінюється значення в застосунку X-Sense.

### Експлуатаційні примітки
- Після налаштування перевірте, що назви пристроїв і кімнати збігаються із застосунком X-Sense.
- Якщо тривога, вимкнення звуку або LED не змінюються одразу, зачекайте наступне MQTT повідомлення або оновлення стану.
- Для станцій SBS50 перевіряйте і стан станції, і окремі дочірні пристрої.
- Для XS01-WX увесь стан може повідомлятися безпосередньо на пристрої, навіть без окремого дочірнього пристрою в обліковому записі.
- Для камер створені сутності залежать від можливостей, які хмара X-Sense повертає для цього облікового запису.
- Якщо сутність не створена, спочатку порівняйте значення із застосунком X-Sense і додайте діагностику.
- Інтеграція призначена для відображення та керування підтримуваними функціями, а не для заміни сполучення або керування пристроями в застосунку.
