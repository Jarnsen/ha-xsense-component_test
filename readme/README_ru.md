## Обзор
Эта интеграция для Home Assistant позволяет использовать устройства X-Sense в системе умного дома. Она была создана на основе оригинального кода [Theo Snel](https://github.com/theosnel/homeassistant-core/tree/xsense/homeassistant/components/xsense) и опубликована с его разрешения и в сотрудничестве с ним.

Эта интеграция HACS активно поддерживается для пользователей, которым нужна более широкая поддержка устройств X-Sense в Home Assistant. Она регулярно обновляется, добавляя новые функции, поддержку устройств и исправления по сообщениям пользователей.

## Функции
- Интеграция различных устройств X-Sense в Home Assistant.
- Поддержка автоматизаций на основе данных датчиков X-Sense.
- Поддержка базовых станций, датчиков дыма, датчиков угарного газа, тепловых датчиков, датчиков протечки, гигрометров, датчиков двери, датчиков движения, света, клавиатур, почтовых ящиков, устройств прослушивания, камер и других поддерживаемых устройств, когда они доступны в учетной записи X-Sense.
- Простая установка через HACS (Home Assistant Community Store).

## Требования
- Рабочий сервер Home Assistant (рекомендуется последняя версия).
- Учетная запись X-Sense с поддерживаемыми устройствами.
- Для установки интеграции необходимо установить HACS в Home Assistant.

## Подготовка
Перед установкой интеграции необходимо выполнить несколько подготовительных шагов:

- **Создайте вторую учетную запись в приложении X-Sense (для использования с Home Assistant)**: Так как невозможно одновременно войти в приложение и в Home Assistant с одной и той же учетной записью, мы рекомендуем использовать отдельную учетную запись для Home Assistant. Это предотвратит постоянный выход из приложения или Home Assistant. Дополнительная учетная запись обеспечивает плавную интеграцию и непрерывное использование без повторных отключений.

- **Поделитесь поддерживаемыми устройствами из основной учетной записи с учетной записью Home Assistant**: Используйте приложение X-Sense, чтобы поделиться **только поддерживаемыми устройствами** с вновь созданной учетной записью. Таким образом, вы сможете легко использовать интеграцию в Home Assistant, продолжая управлять устройствами через свою основную учетную запись.

## Поддерживаемые устройства
Интеграция поддерживает разные устройства X-Sense. Доступные сущности зависят от полей, которые сообщает конкретное устройство и учетная запись. Подтвержденные семейства и модели:
- **Базовая станция (SBS50)**: Центральный хаб для устройств X-Sense.
- **Тепловой датчик (XH02-M)**: Обнаруживает необычно высокую температуру.
- **Датчик угарного газа (XC01-M; XC04-WX)**: Обнаруживает опасную концентрацию CO.
- **Датчик дыма (XS01-M; XS01-WX; XS03-WX; XS0B-MR и связанные RF/iR-модели)**: Раннее обнаружение дыма.
- **Комбинированный датчик CO и дыма (SC07-WX; XP0A-MR и связанные XP/SC-модели)**: Обнаруживает CO и дым.
- **Датчик протечки воды (SWS51)**: Обнаруживает воду в нежелательных местах.
- **Гигрометр-термометр (STH51, STH0A, STH0B, STH0C)**: Отслеживает температуру и влажность.
- **Датчик двери (SDS0A)** и **датчик движения (SMS0A)**: Отображаются, когда X-Sense сообщает соответствующее состояние.
- **Камера (SSC0A, SSC0B)**: Показывает сущности камеры, миниатюры, URL live stream, диагностику и настройки на основе поведения Android-приложения, если это поддержано устройством и учетной записью.
- **Другие устройства станции**: Свет, клавиатура, почтовый ящик, устройство прослушивания, сигнализация подъездной зоны, умное устройство доставки, пульт и данные радона отображаются, когда API сообщает поддерживаемые поля.

### Доступные сущности и действия
Интеграция создает сущности Home Assistant только для полей, которые реально присутствуют в облаке X-Sense, сообщениях MQTT shadow или API камер, согласованных с поведением Android-приложения. В зависимости от устройства это может включать:

- Бинарные сенсоры для alarm, mute, end-of-life, AC-break, water alarm, temperature alarm, charging, motion, door, armed, warning, reminder, light, PIR и состояния keypad.
- Сенсоры батареи, сигнала RF, сигнала Wi-Fi, прошивки, температуры, влажности, уровня CO, пикового CO, громкости тревоги, громкости голосовых сообщений, громкости короткого сигнала, громкости напоминаний, порогов предупреждений, таймеров отключения звука, читаемых меток времени, часового пояса и других диагностических данных.
- Supported camera setup and tuning controls are exposed in Home Assistant when the X-Sense app reports that the feature and account support it.
- Кнопки test, mute, fire-drill и camera wake для моделей, где приложение X-Sense предоставляет соответствующее действие.

Некоторые сущности являются диагностическими или конфигурационными и группируются так в Home Assistant. Если устройство не сообщает конкретное поле или приложение X-Sense помечает функцию как неподдерживаемую для этого устройства/аккаунта, соответствующая сущность не создается. Привязка, удаление, совместный доступ, аккаунт, платежи, обновление прошивки, форматирование SD-карты и другие административные действия остаются в приложении X-Sense.

## Живой просмотр камеры и AI-уведомления
Проще всего импортировать включенный blueprint кнопкой ниже, выбрать `Motion` или доступный `AI Detection` и при необходимости изменить действие уведомления.

[![Импортировать blueprint](https://my.home-assistant.io/badges/blueprint_import.svg)](https://my.home-assistant.io/redirect/blueprint_import/?blueprint_url=https%3A%2F%2Fraw.githubusercontent.com%2FJarnsen%2Fha-xsense-component_test%2Fmain%2Fblueprints%2Fautomation%2Fxsense%2Fcamera_ai_notification.yaml)

Motion и AI Detection — разовые события, а не состояния включено/выключено. Для ручных автоматизаций используйте `event.received`; `event_type` нужен только для фильтрации типов вроде `person`, `pet`, `vehicle`, `package`, `other` или `ai_detection`.

Пример автоматизации:

```yaml
alias: "Notify when X-Sense detects a person"
triggers:
  - trigger: event.received
    target:
      entity_id: event.front_camera_ai_detection
    options:
      event_type:
        - person
actions:
  - action: notify.mobile_app_phone
    data:
      message: "X-Sense camera detected a person."
```

## Примеры автоматизаций
С помощью этой интеграции можно создать различные автоматизации. Вот несколько примеров:

### Пример 1: Предупреждение о температуре
Если температура на термометре X-Sense становится слишком высокой, отправляется уведомление:

```yaml
automation:
  - alias: "Предупреждение о температуре X-Sense"
    trigger:
      platform: numeric_state
      entity_id: sensor.xsense_temperature
      above: 30
    action:
      service: notify.notify
      data:
        message: "Температура превышает 30 градусов!"
```

### Пример 2: Сигнализация датчика протечки воды
Когда датчик протечки воды обнаруживает воду, активируется предупреждение:

```yaml
automation:
  - alias: "Сигнализация датчика протечки воды"
    trigger:
      platform: state
      entity_id: binary_sensor.xsense_waterleak
      to: "on"
    action:
      service: notify.notify
      data:
        message: "Обнаружена утечка воды!"
```

## Полная справка

### Аккаунт и установка
- Используйте отдельный аккаунт X-Sense для Home Assistant.
- Из основного аккаунта X-Sense делитесь только поддерживаемыми устройствами.
- Сопряжение, удаление, общий доступ, прошивки, аккаунт, платежи и управление SD-картой остаются в приложении X-Sense.

### Обновления и использование API
- Быстрые изменения состояния приходят через сообщения MQTT shadow.
- Облачные запросы используются для входа, обнаружения устройств, данных камер и восстановления состояния.
- Периодический опрос служит только резервом, если пропущено live-обновление.

### Сущности, камеры и диагностика
- Сущности создаются только для полей, которые X-Sense действительно сообщает.
- Сущности и элементы управления камерой создаются только тогда, когда API, согласованное с Android-приложением, сообщает поддержку для этого аккаунта и модели.
- Если значения нет, сначала сравните с приложением X-Sense, затем приложите диагностику и соответствующие журналы Home Assistant.

## Контрольный список устройств и сущностей

### Основные семейства устройств
- Базовая станция SBS50, дымовые датчики XS, датчики CO XC, комбинированные датчики SC/XP, тепловые датчики XH, датчики воды SWS, датчики температуры/влажности STH, датчики дверей SDS, датчики движения SMS, камеры SSC и другие сообщаемые семейства X-Sense обрабатываются, когда API предоставляет их поля.

### Поля состояния
- Тревога, отключение звука, батарея, сигнал RF/Wi-Fi, температура, влажность, CO, вода, движение, дверь, свет, напоминания, предупреждения и читаемые метки времени появляются только если X-Sense их сообщает.

### Управление и отчёты
- Переключатели, списки выбора, числовые поля и кнопки создаются только для записываемых настроек и действий, которые предоставляет устройство/аккаунт.
- Хороший отчёт об ошибке включает точную модель, версию интеграции, диагностику, журналы и сведения о том, меняется ли значение правильно в приложении X-Sense.
